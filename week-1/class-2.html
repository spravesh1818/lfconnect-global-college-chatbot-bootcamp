Minimal API Call

Notebook: Section 8 — run the next code cell.

from groq import Groq

client = Groq(api_key=os.environ["GROQ_API_KEY"])
MODEL = "llama-3.3-70b-versatile"

response = client.chat.completions.create(
    model=MODEL,
    messages=[{"role": "user", "content": "What is a chatbot?"}],
)
print(response.choices[0].message.content)

Verify the latest model name on Groq docs — model IDs change over time.

Request Anatomy

FieldPurpose

messagesConversation history (roles + content)
temperatureCreativity (0 = focused, 1+ = varied)

Example: System Changes the Answer

Notebook: Section 9a — same user question, two system prompts.

Same user message — only the system instruction changes (tutor vs comedian).

QUESTION = "Explain what a chatbot is in two sentences."

chat([
    {"role": "system", "content": "You are a patient programming tutor."},
    {"role": "user", "content": QUESTION},
])

chat([
    {"role": "system", "content": "You are a witty comedian. Keep it short."},
    {"role": "user", "content": QUESTION},
])

The model did not get smarter — the instruction changed.

Example: Assistant = Memory

Notebook: Section 9b — append prior turns before the next question.

messages = [
    {"role": "system", "content": "You are a helpful assistant."},
    {"role": "user", "content": "My favorite color is blue."},
]
first_reply = chat(messages)
messages.append({"role": "assistant", "content": first_reply})
messages.append({"role": "user", "content": "What is my favorite color?"})
print(chat(messages))

Without the assistant turn, the model would not remember what you said earlier.

The chat(messages) Pattern

Notebook: Section 9c — one helper for all role patterns; Sections 12–15 use ask() wrappers.

def chat(messages: list) -> str:
    response = client.chat.completions.create(
        model=MODEL,
        messages=messages,
        temperature=0.7,
    )
    return response.choices[0].message.content

def ask(question: str) -> str:
    return chat([{"role": "user", "content": question}])

Rate Limits & Latency

Rate limits

  • Requests per minute (RPM)
  • Tokens per minute (TPM)
  • HTTP 429 when exceeded

Latency

  • Time to first token
  • Total generation time
  • Larger models = slower, often smarter

Robust API Call Pattern

Notebook: Section 15 — ask_safe().

def ask_safe(question: str) -> str:
    try:
        response = client.chat.completions.create(
            model=MODEL,
            messages=[{"role": "user", "content": question}],
        )
        return response.choices[0].message.content
    except Exception as e:
        return f"Sorry, something went wrong: {e}"
