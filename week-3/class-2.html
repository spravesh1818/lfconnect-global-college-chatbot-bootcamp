The Core Trade: Exact vs Approximate

Exact (k-NN)

  • Compare against everything
  • 100% correct top-k
  • Cost grows linearly with N

Approximate (ANN)

  • Smart data structures skip most vectors
  • ~95–99% of the true top-k
  • 10–1000× faster, sub-linear

RAG retrieves context for an LLM that reads 3–5 chunks. Missing the true #7 result almost never changes the answer — ANN is the right trade.

FAISS

Facebook AI Similarity Search — Meta's open-source C++ library (with Python bindings) for vector indexing. The industry workhorse since 2017.

  • A library, not a database — lives inside your Python process
  • Battle-tested at billion-vector scale, optional GPU acceleration
  • Many index families — we'll master three: Flat, IVF, HNSW

Notebook · Section 4

Baseline: The Flat Index (Exact)

d = 384                                  # vector dimension
index = faiss.IndexFlatIP(d)             # IP = inner product

faiss.normalize_L2(embs)                 # normalize → IP == cosine
index.add(embs)                          # (N, 384) float32
print(index.ntotal)                      # N vectors stored

scores, ids = index.search(query_vec, k=5)   # both (1, 5)
  • IndexFlatIP + L2-normalized vectors = exact cosine search
  • Same results as your NumPy loop — but C++ speed and a stable API
  • Returns positions (ids), not text — remember that

Notebook · Section 6

IVF: Cluster First, Search Less

Train: k-means → 100 cells→ Assign each vector to a cell→ Query: scan only nprobe cells 
nlist = 100                                   # number of clusters
quantizer = faiss.IndexFlatIP(d)
index = faiss.IndexIVFFlat(quantizer, d, nlist, faiss.METRIC_INNER_PRODUCT)

index.train(embs)          # learn the 100 cluster centers
index.add(embs)
index.nprobe = 10          # search 10 of 100 cells → ~10× less work

nprobe is your quality dial: 1 = fastest/riskiest, nlist = exact-but-slow.

Notebook · Section 7

HNSW: A Graph You Can Ski Down

Hierarchical Navigable Small World — every vector becomes a graph node linked to its neighbors, with express layers on top like a skip list. Queries greedily hop toward the target.

index = faiss.IndexHNSWFlat(d, 32, faiss.METRIC_INNER_PRODUCT)  # 32 links/node
index.hnsw.efSearch = 64        # exploration width at query time
index.add(embs)                 # no training step needed
  • Best recall/speed trade at query time — the default in most modern vector DBs (including Qdrant)
  • Cost: more RAM (the graph) and slower inserts

Recall@k — Measuring "Good Enough"

true_ids = exact_index.search(queries, k)[1]     # ground truth
ann_ids  = ivf_index.search(queries, k)[1]       # approximate

recall = np.mean([
    len(set(t) & set(a)) / k
    for t, a in zip(true_ids, ann_ids)
])
  • Recall@k — of the true top-k, what fraction did the ANN index return?
  • 0.95 means: on average, 9.5 of the true top-10 survive
  • This is how you tune nprobe / efSearch with evidence, not vibes

The Metadata Problem

FAISS stores vectors only. index.search() gives you row numbers — it's on you to map them back to text and source.

chunk_store = [                       # position i ↔ vector i in the index
    {"text": c["text"], "source": c["source"]} for c in all_chunks
]

scores, ids = index.search(q, k=3)
for score, i in zip(scores[0], ids[0]):
    hit = chunk_store[i]
    print(f"{score:.3f}  [{hit['source']}]  {hit['text'][:70]}")

Want "search only PDF chunks" or "price < £20"? With FAISS you filter after search, by hand. A real vector DB does this natively — that's Class 3.

Persistence: Embed Once, Load Forever

faiss.write_index(index, "week3_chunks.faiss")     # vectors
json.dump(chunk_store, open("chunk_store.json", "w"))  # texts + sources

# next session / other notebook:
index = faiss.read_index("week3_chunks.faiss")
chunk_store = json.load(open("chunk_store.json"))
  • Embedding 100K chunks costs minutes; searching costs milliseconds — never re-embed what you can reload
  • Index file + store file travel together, always in the same order

Choosing an Index — Cheat Sheet

SituationUseWhy
< 100K vectors (your project!)IndexFlatIPExact, instant, zero tuning
100K – 10M, RAM is fineHNSWBest query-time recall/speed
10M+, RAM is tightIVF (+PQ compression)Controls memory, tunable via nprobe
Need filters, CRUD, a serverVector DB (Qdrant…)FAISS is a library, not a database

Rule of thumb: start Flat, graduate when latency hurts. Premature ANN is premature optimization.
