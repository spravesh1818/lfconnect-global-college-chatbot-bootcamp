We use Qdrant: QdrantClient(":memory:") runs the full API inside Colab — the same code later points at a real server with one line changed.

Qdrant's Three Nouns

  • Collection — a named set of vectors with one config (size=384, distance=Cosine). Like a SQL table.
  • Point — one entry: id + vector + payload. Like a row.
  • Payload — arbitrary JSON riding along with the vector: text, source, page, price… Like the other columns — and filterable.
PointStruct(
    id=42,
    vector=[0.12, -0.87, ...],                       # 384 floats
    payload={"text": "Automobile repair pricing...",
             "source": "sample.pdf", "page": 3},
)

from qdrant_client import QdrantClient
from qdrant_client.models import Distance, VectorParams, PointStruct

client.create_collection(
    collection_name="bootcamp_chunks",

Notebook · Section 6

Filtered Search — Meaning AND Metadata

from qdrant_client.models import Filter, FieldCondition, MatchValue, Range

hits = client.query_points(
    collection_name="bootcamp_chunks",
    query=q,
    query_filter=Filter(must=[
        FieldCondition(key="source", match=MatchValue(value="sample.pdf")),
    ]),
    limit=3,
).points

# or on your book dataset:  Range(key="price_gbp", lte=20.0)

Remember Class 1: embeddings can't do "under £20". Filters + vectors together can — this is the fix.

The Retrieval Pipeline

Question→ Embed→ Qdrant top-k→ Context block→ Groq LLM 
def retrieve(question: str, k: int = 3) -> list[dict]:
    q = model.encode(question).tolist()
    hits = client.query_points("bootcamp_chunks", query=q, limit=k).points
    return [
        {"text": h.payload["text"], "source": h.payload["source"],
         "score": h.score}
        for h in hits
    ]
