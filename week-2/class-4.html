Why Collect Data?

Scrape / PDF→ Clean text→ Chunk→ Embed (Week 3)

Your research node can search the open web. But docs only had a made-up placeholder sentence — until we feed it your data.

Web Scraping Ethics

  • Check robots.txt — is scraping allowed?
  • Use permitted practice sites (toscrape.com is built for learning)
  • Add delays between requests — don't hammer servers
  • Respect copyright — scraped text is for your course project only
  • Never scrape login-protected or personal data

In production: get permission, use official APIs when available.

Extract & Save Blog Posts

posts = []
for card in soup.select("article"):
    title = card.select_one("h3").get_text(strip=True)
    date = card.select_one("time")
    summary = card.select_one("p")
    posts.append({
        "title": title,
        "date": date.get("datetime", "") if date else "",
        "summary": summary.get_text(strip=True) if summary else "",
    })

json.dump(posts, open("data/python_blog_posts.json", "w"), indent=2)

Notebook · Section 12

Chunking for Embeddings

def chunk_text(text: str, chunk_size: int = 500, overlap: int = 50) -> list[str]:
    chunks = []
    start = 0
    while start < len(text):
        end = start + chunk_size
        chunks.append(text[start:end].strip())
        start = end - overlap
    return [c for c in chunks if c]

chunks = chunk_text(full_text)
print(f"{len(chunks)} chunks, first: {chunks[0][:100]}...")

Week 3: each chunk becomes a vector in FAISS / Qdrant.

Why Chunk Size Matters

Chunk sizeProsCons
Small (200–400)Precise retrievalMay lose context
Medium (500–800)Balanced (good default)—
Large (1000+)More context per chunkNoisy embeddings

Overlap prevents sentences from being cut in half at boundaries.
