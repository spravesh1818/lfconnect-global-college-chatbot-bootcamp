Prompting Is the Interface

The model is fixed. Your prompt is the user interface.

Same LLM+ Weak prompt→ Generic reply
Same LLM+ Strong prompt→ Exam-ready output

You cannot swap the model in production every week — but you can improve the prompt today.

Bad vs Good — Same Topic

Bad prompt

Tell me about photosynthesis.

Result: long essay, no exam focus, random depth

Good prompt

Summarize photosynthesis for a 45-min exam.
Bullets: definition, inputs/outputs,
light vs dark reactions, one common trap.

Result: scannable, revision-ready, predictable format

What Bad Prompts Cost

  • Hallucinations — model fills gaps when instructions are vague
  • Wrong format — you re-prompt, re-copy, re-edit manually
  • Wasted tokens — long rambling answers cost money and time
  • Re-prompt loops — "no, shorter" / "no, as bullets" / "try again"
  • User distrust — inconsistent answers erode confidence

In a RAG chatbot, a bad prompt wastes good retrieved data.

Act 2

Anatomy of a Prompt

Roles · Anti-patterns

Prompt ≠ Model ≠ API

Your prompt→ API (Groq)→ Model (LLaMA)→ Response
  • Prompt — what you control most directly
  • API — transport, roles, temperature, token limits
  • Model — weights, knowledge cutoff, reasoning ability

Five Building Blocks

1. InstructionsWhat to do
2. ContextBackground, audience, exam type
3. ExamplesFew-shot input/output pairs
4. FormatBullets, JSON, flashcards, steps
5. ConstraintsMax length, scope limits, "do NOT" rules

Every strong prompt in this class uses at least 3 of these 5.

Negative Instructions

Tell the model what not to do — prevents predictable failures.

Do NOT invent citations or page numbers.
Do NOT answer beyond revision scope — say "I don't have that source."
Do NOT exceed 150 words unless the user asks for more.

Especially critical for RAG: "Only use the provided context. If the answer is not in the context, say so."

Prompt Roles

RoleWho writes itPurpose
systemDeveloperGlobal behavior, tone, rules, format defaults
userHumanQuestions, tasks, new input each turn
assistantModel (past turns)Memory and few-shot demonstrations

ChatGPT "Custom Instructions" ≈ system · Your messages ≈ user · Prior replies ≈ assistant

How Roles Shape Multi-Turn Chat

[
  {"role": "system", "content": "You are an exam prep tutor."},
  {"role": "user", "content": "Topic: Mitosis"},
  {"role": "assistant", "content": "- Term: Mitosis\n- Definition: ..."},
  {"role": "user", "content": "Topic: Photosynthesis"},
]

The assistant turn teaches flashcard format before the real task — few-shot via roles.

Anti-Pattern: Rules in User Message

Bad: Putting system rules in the user turn

User: You must always answer in bullet points. What is mitosis?

May work once — often ignored by turn 2 when new user messages arrive.

Fix: Move rules to system; keep user for the actual question only.

Anti-Pattern: Few-Shot in System

Stuffing examples into the system message sometimes works — but assistant turns are clearer.

Fragile

Long system block with embedded "Example 1… Example 2…"

Clear

System = rules · User/Assistant turns = demonstrations

Assistant Role — Two Jobs

Few-shot demoFabricated example showing desired format
Conversation memoryReal prior replies in multi-turn chat

Same role, different intent. In LangChain (Class 2): ("ai", ...) for demos, MessagesPlaceholder for history.

Act 3

Four Techniques

Definition → bad → good → second domain

Zero-Shot — Definition

Instruction only — no examples. The model relies on pre-training.

  • Mental model: "The model already knows this — just tell it the job"
  • Reach for it when: Quick drafts, well-known topics, simple Q&A

Zero-Shot — Bad Example

Explain photosynthesis.

Why it fails: No audience, no format, no scope → rambling essay, wrong depth for an exam.

Zero-Shot — Good (Biology)

Summarize the key concepts of photosynthesis I need for a 45-minute exam.
Use bullet points: definition, inputs/outputs, light vs dark reactions, one common exam trap.

Why it works: Context (exam), format (bullets), scope (four sections).

Zero-Shot — Good (History)

Summarize the main causes of WWI for a 10-mark essay.
4 bullets — one historian's name per cause. Max 120 words.

Domain-agnostic: Same technique, different subject. Constraints lock the output.

Few-Shot — Definition

1–3 demonstrations before the real task. The model mirrors structure.

  • Mental model: "Show, don't tell"
  • Reach for it when: Format must be exact — flashcards, JSON, classification

Few-Shot — Bad Example

User: Here's an example: mitosis is cells dividing. Now do photosynthesis.
Assistant: (inconsistent format — sometimes prose, sometimes bullets)

Why it fails: Example doesn't demonstrate the target format — model copies messiness.

Few-Shot — Good (Biology)

User: Topic: Mitosis
Assistant:
- Term: Mitosis
- Definition: Cell division producing two identical diploid cells
- Exam tip: Don't confuse with meiosis (4 haploid cells)

User: Topic: Photosynthesis

Model mirrors Term / Definition / Exam tip structure.

Few-Shot — Good (History)

User: Cause #1: Militarism
Assistant:
- Event: Anglo-German naval race
- Cause: Arms competition increased tension
- Evidence: Fischer — Germany sought world power status
- Counter: Some historians emphasize domestic politics

User: Cause #2: Imperialism

Chain-of-Thought — Definition

Model shows intermediate reasoning steps before the final answer.

  • Mental model: "Show your working" — like exam mark schemes
  • Reach for it when: Math, logic, multi-step analysis, worked problems

Chain-of-Thought — Bad Example

What is the answer to this Calvin cycle problem? A plant fixes 12 CO2...

Why it fails: Model jumps to final answer — no auditable steps, can't spot where reasoning broke.

Chain-of-Thought — Good (Biology)

A plant fixes 12 molecules of CO2 in one hour via the Calvin cycle.
Walk through each stage (carbon fixation, reduction, regeneration).
Show your reasoning at each step, then state how many G3P molecules are produced.

Chain-of-Thought — Good (History)

Why did the alliance system escalate a regional conflict into WWI?
Walk through events from June 28 to August 4, 1914, step by step.
At each step, state which alliance clause was triggered.

Zero-Shot CoT — Definition

Add "Let's think step by step" — no examples, but elicits reasoning.

  • Mental model: Magic phrase unlocks deliberation
  • Reach for it when: You need analysis but lack good few-shot examples

Zero-Shot CoT — Bad Example

Compare photosynthesis and cellular respiration. Which is more efficient?

Why it fails: No trigger phrase → shallow one-paragraph comparison, no revision questions.

Zero-Shot CoT — Good (Biology)

Let's think step by step.
Compare photosynthesis and cellular respiration.
Which process is more efficient at converting light energy to usable chemical energy, and why?
End with three revision questions I should practice.

Zero-Shot CoT — Good (History)

Let's think step by step.
Compare militarism and imperialism as causes of WWI.
Which had greater impact in July 1914, and why?
End with 3 essay thesis statements I could defend.

When NOT to Use Each Technique

TechniqueDon't use when…
Zero-shotFormat must be exact every time
Few-shotExamples are inconsistent or misleading
Chain-of-thoughtSimple factual lookup ("What year did WWI start?")
Zero-shot CoTYou need auditable, numbered steps (use explicit CoT)

Act 4

Frameworks

Frameworks & format control

CO-STAR Framework

LetterMeaningExam prep example
CContext"45-min Biology exam on photosynthesis"
OObjective"Create revision flashcards"
SStyle"Concise, student-friendly"
TTone"Encouraging tutor"
AAudience"First-year university student"
RResponse"Bullet flashcards with exam tips"

Full CO-STAR Prompt (Copy-Paste)

Context: I am revising for a 45-minute Biology exam on photosynthesis.
Objective: Help me create revision flashcards I can memorize tonight.
Style: Concise, student-friendly, no jargon without explanation.
Tone: Encouraging tutor, not condescending.
Audience: First-year university student, strong in humanities, weaker in sciences.
Response format: For each topic, return Term / Definition (max 2 sentences) / Exam tip.

Do NOT invent facts beyond standard A-level/intro university Biology.

CRISP Framework

LetterMeaningExam prep example
CContext"Calvin cycle, 12 CO2 molecules fixed"
RRole"You are a Biology exam coach"
IInstructions"Walk through each stage step by step"
SSpecifics"Name intermediates; count G3P output"
PParameters"Max 200 words; numbered steps"

Full CRISP Prompt (Copy-Paste)

Context: A plant fixes 12 CO2 molecules per hour via the Calvin cycle.
Role: You are a Biology exam coach preparing me for a problem-style question.
Instructions: Walk through carbon fixation, reduction, and regeneration.
Specifics: Name key intermediates (RuBP, G3P). Show reasoning at each step.
Parameters: Numbered steps, max 200 words, end with final G3P count.

Do NOT skip steps or jump to the final answer.

Format Control for Code Pipelines

When your chatbot must parse output, request JSON:

{
  "term": "Photosynthesis",
  "definition": "Process converting light energy to chemical energy in glucose",
  "exam_tip": "Don't confuse light reactions (thylakoid) with Calvin cycle (stroma)"
}

Also: Markdown tables, numbered steps, flashcard bullets — match format to downstream use.

Choosing a Framework

  • CO-STAR — tone, audience, creative study coaches
  • CRISP — precise, step-by-step, worked problems
  • Few-shot alone — when format is easier to show than describe
  • Combine — CO-STAR system prompt + few-shot assistant demos (most production bots)

ChatGPT Workflows

Iterate before you code

Prompt Laboratory Mindset

ChatGPT

Cheap, fast iteration. Throw away 10 drafts. Keep the best.

LangChain (Class 2)

Lock the winning prompt into templates. Ship to users.

Never code a prompt you haven't tested in conversation first.

Iteration Loop

Draft→ 3 test inputs→ Score→ Tighten→ Edge cases→ Export

Test inputs: one easy, one hard, one off-topic. Score: accuracy, format, length, hallucination.

Custom GPTs — 4 Steps

  1. Name + role — "Biology Exam Coach"
  2. Instructions — paste your CO-STAR system prompt
  3. Conversation starters — "Quiz me on photosynthesis", "Calvin cycle problem"
  4. Adversarial test — vague, off-topic, and trick questions (Task 4)

Task 3 — Build Your Exam Coach

10 min · solo

Write a full CO-STAR system prompt for an exam-prep coach in your subject.

Paste into ChatGPT Custom Instructions. Test with 2 questions.

Task 3 — Peer Review Rubric

  • Would you trust this coach for your exam?
  • Does every CO-STAR letter appear explicitly?
  • Is output format specific (not just "be clear")?
  • Is there at least one Do NOT rule?
  • Did 2 test questions produce usable study material?

Task 4 — Adversarial Test

5 min · pairs

Try to break your Task 3 prompt with:

  • Vague: "help me study"
  • Off-topic: "write me a poem"
  • Trick: "what's the answer to question 5 on my exam?"

Task 4 — Debrief: Guardrails = RAG Prep

Which failures happened? Add one constraint to fix the worst failure.

Examples: "If the question is off-topic, redirect to exam revision." · "Never claim to know specific exam questions." · "If input is vague, ask one clarifying question."

This is exactly how we build safe RAG bots in Weeks 3–4.

Discussion: RAG + Prompting

  • System prompt — "Only answer from provided context" (Week 3–4)
  • Few-shot — lock citation format when displaying sources
  • CoT — reason over multiple retrieved chunks
  • Do NOT constraints — from Task 4 → production guardrails
  • Zero-shot — quick summaries of a retrieved passage
